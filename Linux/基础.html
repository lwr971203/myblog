    # 系统管理

    # init

    System V init 的工具，用于切换系统运行级别

    init <运行级别>

    7 种运行级别：

    • 0：关机
    • 1：单用户维护模式
    • 2：多用户命令行模式，无网络
    • 3：多用户命令行模式，有网络
    • 4：未使用
    • 5：多用户图形界面
    • 6：重启

    查看当前系统的运行级别

    runlevel

    在现代 linux 发行版中，System V init 已经被 systemd 所取代，但是 System V init 的绝大部分命令仍然被保留了下来

    # service

    System V init 的工具，用于管理指定的服务

    启动

    service <服务> start

    停止

    service <服务> stop

    重启

    service <服务> restart

    显示状态信息

    service <服务> status

    重新加载配置文件但不中断

    service <服务> reload

    # systemctl

    systemd 的工具，用于管理指定的 unit

    在 systemd 中，unit 是系统资源的抽象，服务也属于一种 unit

    启动

    systemctl start <unit>

    停止

    systemctl stop <unit>

    重启

    systemctl restart <unit>

    查看状态信息

    systemctl status <unit>

    重新加载配置文件但不中断

    systemctl reload <unit>

    设置开机自启

    systemctl enable <unit>

    禁用开机自启

    systemctl disable <unit>

    是否已经运行

    systemctl is-active <unit>

    是否已经设置开机自启

    systemctl is-enabled <unit>

    屏蔽（使得该 unit 不得以任何方式启动）

    systemctl mask <unit>

    取消屏蔽

    systemctl unmask <unit>

    列出所有已经启动的 unit

    systemctl list-units [选项]

    选项：

    • --all：列出时包括未启动的 unit
    • --type=service：仅列出服务类 unit

    列出所有已经安装的 unit 它们的开机自启状态

    systemctl list-unit-files

    重新加载 systemd 以及所有 unit 的配置文件，但是不中断

    systemctl daemon-reload

    获取系统的默认目标（每次开机时的目标）

    systemctl get-default

    在 systemd 中，目标是一种系统运行模式，用来组织和协调不同 unit 的启动过程，每个 unit 都必须在系统到达该目标时处于预配置好状态

    ==常见的目标==：

    • multi-user.target：多用户命令行模式（对应运行级别 3）
    • graphical.target：多用户图形界面（对应运行级别 5）
    • poweroff.target：关机（对应运行级别 0）
    • reboot.target：重启（对应运行级别 6）
    • rescue.target：单用户维护模式（对应运行级别 1）
    • emergency.target：单用户维护模式（更低级别的维护模式）

    设置系统的默认目标

    systemctl set-default <目标>

    立即切换到指定的目标（不会改变默认目标）

    systemctl isolate <目标>

    发送终止类信号

    systemctl kill [选项] <unit>

    选项：

    • --signal=SIGTERM：优雅停止 unit（这是默认）
    • --signal=SIGKILL：强制停止 unit
    • --kill-who=main：停止的是 unit 的主进程
    • --kill-who=control：停止的是 unit 的控制进程
    • --kill-who=all：停止的是 unit 的主进程以及相关的所有进程
    • --now：立即执行

    发送终止类信号这个命令实际上并不好用，没有 systemctl stop 这个命令（也是优雅停止）好用，因为有一些 unit 并不会理会这些信号

    查看 unit 的配置信息

    systemctl cat <unit>

    编辑 unit 的配置信息

    systemctl edit <unit>

    列出 unit 的运行属性

    systemctl show <unit>

    显示 unit 的帮助手册（如果有）

    systemctl help <unit>

    # 关机

    关机（断电）

    poweroff // 立即关机
systemctl poweroff // 立即关机
init 0 // 立即关机
shutdown -h now // 立即关机

shutdown -h <分钟> [msg] // 在指定分钟后关机，并显示关机消息
shutdown -h <hh::mm> [msg] // 在指定时间点关机，并显示关机消息

    停机（不断电）

    halt // 停机不断电（谨慎使用，会导致系统当场卡住）

    重启

    reboot // 立即重启
systemctl reboot // 立即重启
init 6 // 立即重启
shutdown -r now // 立即重启

shutdown -r <分钟> [msg] // 在指定分钟后重启，并显示关机消息
shutdown -r <hh::mm> [msg] // 在指定时间点重启，并显示关机消息

    取消所有关机或重启计划

    shutdown -c

    将数据同步到磁盘

    sync

    在 linux 中，对磁盘的采用了预读入缓输出的机制，所以在关机之前，建议使用 sync 确保将内存中的数据同步到磁盘上

    # 帮助

    # man

    显示 linux 的手册

    man [选项] [章节] <关键词>

    除了 man 命令，还有一个变种命令 info，它显示的内容通常比 man 更多

    man 手册分为九章：

    1. 可执行程序或 shell 命令
    2. 系统调用（内核提供的函数）
    3. 库函数
    4. 特殊文件（通常位于 /dev ）
    5. 文件格式和规范（例如 man 5 passwd）
    6. 游戏
    7. 杂项（包括宏和规范）
    8. 系统管理命令（通常只针对 root 用户）
    9. 内核例程（非标准）

    选项：

    • -a：显示所有找到关键词的章节，而不只是默认找到的那个章节
    • -k：显示所有与关键词相关的标题
    • -f：显示简短的描述信息，完全等价于 whatis 命令
    • -w：显示帮助信息所存储的位置

    # help

    显示命令中内置的帮助手册（如果有）

    <命令> --help

    # 文件

    # cd

    切换到指定的目录

    cd [绝对路径或相对路径]

    切换到家目录

    cd // 切换到当前用户的家目录
cd ~ // 切换到当前用户的家目录
cd ~<user_name> // 切换到指定用户的家目录

    切换到上一次所在目录

    cd -

    切换到符号链接实际所指向目录的物理路径

    cd -P <符号链接文件>

    如果不使用 -P 选项，默认会切换到符号链接本身的逻辑路径（即 -L 选项）

    # pwd

    打印当前所在的工作路径

    pwd

    打印当前所在符号链接实际所指向目录的物理路径

    pwd -P

    如果不使用 -P 选项，默认会答应符号链接本身的逻辑路径（即 -L 选项）

    # ls

    列出指定文件或目录的名称

    ls [文件或目录]
// 如果不指定，默认就是 .
// 如果是目录，则列出该目录下的所有子项目名称

    包括所有隐藏文件或目录

    ls -a [文件或目录]

    以多列的方式显示详细信息

    ls -l [文件或目录]

    以特别的样式显示文件的类型

    ls -F [文件或目录]

    包括所有隐藏文件或目录，但是不包括当前目录和父目录

    ls -A [文件或目录]

    显示目录自身的信息，而不是该目录下的所有子项目

    ls -d [目录]

    显示 inode 号

    ls -i [文件或目录]

    按修改时间从近到晚排序

    ls -t [文件或目录]

    按文件大小从大到小排序

    ls -S [文件或目录]

    按文件拓展名排序

    ls -X [文件或目录]

    逆序

    ls -r [文件或目录]

    递归

    ls -R [文件或目录]

    以人容易阅读的方式显示

    ls -h [文件或目录]

    # touch

    修改文件的 atime 和 mtime（如果不存在则创建文件）

    touch [文件] // 更新 atime 和 mtime 为基准时间
touch -t <yyyymmddhhMM.ss> <文件> // 更新 atime 和 mtime 为指定的时间
touch -d <时间字符串> <文件> // 更新 atime 和 mtime 为指定的时间

    基准时间未提供（未提供 -r 选项）默认就是当前时间

    时间字符串支持的格式：

    • "yyyy-mm-dd hh:MM:ss"
    • "+或-n days或months或years" 以基准位置向前或向后偏移 n 天或月或年
    • "n 英文星期" 以基准位置向前偏移 n 个英文星期

    修改文件的 atime 和 mtime（如果不存不会创建文件）

    touch -c [文件] // 更新 atime 和 mtime为基准时间
touch -c -t <yyyymmddhhMM.ss> <文件> // 更新 atime 和 mtime 为指定的时间
touch -c -d <时间字符串> <文件> // 更新 atime 和 mtime 为指定的时间

    修改文件的 atime（如果不存在则创建文件）

    touch -a [文件] // 更新 atime 为基准时间
touch -a -t <yyyymmddhhMM.ss> <文件> // 更新 atime 为指定的时间
touch -a -d <时间字符串> // 更新 atime 为指定的时间

    修改文件的 mtime（如果不存在则创建文件）

    touch -m [文件] // 更新 mtime 为基准时间
touch -m -t <yyyymmddhhMM.ss> <文件> // 更新 mtime 为指定的时间
touch -m -d <时间字符串> <文件> // 更新 mtime 为指定的时间

    提供参考文件的 mtime 作为 touch 的基准时间

    touch -r [参考文件] [文件]

    结合通配符创建多个文件

    touch file{1..10}.sh
